# Contributing

## Purpose

This repository is structured as a production-quality cloud security engineering project. Contributions should preserve repository clarity, implementation traceability, and documentation quality.

## Contribution Expectations

- Keep infrastructure and documentation changes scoped and reviewable
- Preserve AWS and GCP symmetry where practical
- Document design intent before adding implementation details
- Avoid mixing unrelated changes in a single update

## Pull Request Guidance

- Describe the change clearly
- Call out affected cloud scope: AWS, GCP, or shared
- Identify evidence or screenshots that should be updated
- Note any follow-on documentation updates that remain

## Documentation First

Before major implementation work, update the relevant placeholder files in `docs/`, `comparison/`, `architecture/`, or `screenshots/` so the repository remains navigable for reviewers.
